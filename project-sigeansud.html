Android (Java) Symfony 7 API REST MySQL Laragon

Parc animalier SIGEANSUD

Un parc animalier moderne qui doit mettre en place une solution informatique pour gérer l'alimentation de ses animaux de manière traçable et optimisée.

Jusqu'à présent, la gestion des repas se faisait manuellement. Le projet vise à : moderniser le processus, assurer la traçabilité complète, et faciliter le suivi de la santé des animaux par les gestionnaires du parc.

Mission

Objectifs du projet

Créer une solution full-stack pour gérer efficacement l'alimentation des animaux du parc.

📱 Application Mobile

Soignants saisissent en temps réel les heures et quantités de repas donnés aux animaux de leurs espèces affectées.

🖥️ Back-office Web

Gestionnaires administrent les soignants, affectent les espèces, définissent les menus et consultent les historiques de traçabilité complets.

🔗 API REST

Couche métier centralisée fournissant les données aux applications mobile et web, avec synchronisation et gestion complète de la persistance.

📊 Base de données

MySQL en production (Laragon) : modèle organisationnel complet, MCD normalisé, support de la traçabilité et des historiques.

🎯 Traçabilité

Enregistrement complet des repas donnés : heure, quantité, soignant, animal, espèce. Historique consultable et rapports pour les vétérinaires.

⚙️ Gestion des menus

Menus adaptés par espèce avec portions, fréquences et recommandations nutritionnelles. Géré centralement, consultable par les soignants sur l'app mobile.

Structure

Architecture applicative

Frontend - Application Mobile (Android)

  • Développement en Java sous Android Studio
  • Interface intuitive pour saisie rapide des repas
  • Synchronisation automatique avec l'API REST
  • Accès filtré selon les espèces affectées au soignant

Backend - Back-office & API (Symfony 7)

  • Framework Symfony 7 pour robustesse et scalabilité
  • Back-office web pour gestionnaires du parc
  • API REST pour communication avec le mobile
  • Développement local avec Laragon
  • Base de données MySQL en développement

Documentation & Méthodologie

  • Modèle Organisationnel de Traitement (MOT) validé avec les professeurs
  • Spécification complète de l'API REST
  • Diagrammes UML : classes, séquences, MCD
  • Maquettage détaillé des deux applicatifs
  • Plans de tests avec données cohérentes
  • Documentation utilisateur pour soignants et gestionnaires
  • Suivi agile via JIRA/Trello

Décisions

Choix architecturaux

Une architecture réfléchie pour assurer performance, scalabilité et facilité de maintenance.

🔄 API REST centralisée

Tous les clients (web, mobile) passent par l'API. Simplifie la maintenance, assure la cohérence des données et facilite les évolutions futures.

📱 Stockage local SQLite

L'app mobile cache localement les données pour fonctionner en mode offline. Synchronisation intelligente lors de la reconnexion réseau.

🏗️ Symfony 7

Framework moderne, bien documenté et soutenu. Excellent pour construire des APIs REST fiables et des applications web professionnelles.

🏠 Développement local

Environnement de développement avec Laragon pour faciliter le prototypage et les tests en local avant déploiement.

📋 Méthodologie Agile

JIRA/Trello pour organiser les sprints, suivre les tâches par étudiant et assurer un développement itératif et collaboratif.

📊 MOT & diagrammes

Modélisation complète des flux applicatifs avant développement. Validation des choix architecturaux avec l'équipe pédagogique.
